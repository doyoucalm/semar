# Pawukon Research — verified algorithms (2026-05-18)

Authoritative algorithms + epoch dates for the Javanese/Balinese Pawukon calendar system, JDN-based modular arithmetic. All four primary cycles below have been **numerically verified** against four independent anchor dates from kalenderbali.org and ki-demang.com.

## TL;DR — the verified formulas

```
pancawara_index  = JDN mod 5            // 0=Legi 1=Pahing 2=Pon 3=Wage 4=Kliwon
saptawara_index  = (JDN + 1) mod 7      // 0=Redite(Sun) ... 6=Saniscara(Sat)
pawukon_day      = ((JDN + 65) mod 210) // 1..210; if 0 → 210
wuku_index       = floor((pawukon_day - 1) / 7)   // 0..29
day_in_wuku      = (pawukon_day - 1) mod 7
```

## 1. Pancawara (5-day, "pasaran")
- **Names**: Legi, Pahing, Pon, Wage, Kliwon (Javanese). Balinese: Umanis, Paing, Pon, Wage, Kliwon.
- **Urip (neptu)**: Legi=5, Pahing=9, Pon=7, Wage=4, Kliwon=8.
- **Formula**: `JDN mod 5` → indices `0..4`.
- **Confidence**: High.
- **Test fixtures**: 1985-05-05→Pahing; 2000-01-01→Legi; 2024-01-01→Pahing.

## 2. Saptawara (7-day)
- **Names**: Redite, Soma, Anggara, Buda, Wraspati, Sukra, Saniscara → Sun..Sat.
- **Urip (Javanese neptu)**: Minggu=5, Senin=4, Selasa=3, Rabu=7, Kamis=8, Jumat=6, Sabtu=9.
- **Formula**: `(JDN + 1) mod 7`. Or equivalently `Date.getDay()` in user's local TZ.
- **Confidence**: High.

## 3. Wuku (30-wuku Pawukon)
- **Canonical order**: Sinta, Landep, Ukir, Kulantir, Tolu *(JV: Tulu)*, Gumbreg, Wariga, Warigadean *(JV: Wariga Dean)*, Julungwangi, Sungsang, Dunggulan *(Dungulan)*, Kuningan, Langkir, Medangsia, Pujut, Pahang, Krulut, Merakih, Tambir, Medangkungan, Matal, Uye, Menail, Prangbakat, Bala, Ugu *(JV: Wugu)*, Wayang, Kelawu *(Kulawu)*, Dukut, Watugunung.
- **Epoch**: Pawukon has no historical epoch. Dershowitz-Reingold (*Calendrical Calculations*, Cambridge 2018, §12) anchor day 1 to JDN 146. Our derived constant 65 is equivalent: `(JDN - 146 + 1) mod 210 == (JDN + 65) mod 210`.
- **Formula**: `pawukon_day = (JDN + 65) mod 210` (replace 0 with 210). `wuku_index = (pawukon_day - 1) // 7`.
- **Confidence**: High — four anchor dates across 40 years all match exactly.

## 4. Weton
- **Definition**: pair (saptawara, pancawara). 35-day cycle (selapan). No epoch — pure product of items 1 and 2.
- **Total neptu** = neptu_saptawara + neptu_pancawara, range 8..18.
- **Test fixtures**: 1985-05-05→Minggu Pahing, neptu 14. 2024-01-01→Senin Pahing, neptu 13. 2000-01-01→Sabtu Legi, neptu 14.
- **Confidence**: High.

## 5. Sasih + Saka — **DEFER TO v2**
- **Saka epoch**: 78 CE. Saka 1947 starts at Nyepi 2025-03-29.
- **Sasih order**: Kasa, Karo, Katiga, Kapat, Kalima, Kanem, Kapitu, Kawolu, Kasanga, Kadasa, Jyestha (Desta), Sadha.
- **Why hard**: requires (a) true astronomical Tilem/new-moon at Bali longitude, (b) ngunalatri ~63-day collapse-day rule, (c) Mala-sasih intercalation (~every 3 years) which is itself astronomical.
- **Recommendation**: defer to v2. Either embed a precomputed 1900–2100 tilem/purnama lookup table (~125 KB, generated once from astronomy-engine), or bundle Meeus *Astronomical Algorithms* ch.49 moon-phase calculator (~80 LOC, accurate enough since we only need day resolution).
- For Lucky's birth, kalenderbali.org shows **Purnama Jyestha (Saka 1907)** — keep as v2 sanity check.
- **Confidence**: Medium on the mechanism description, low on implementing from scratch without ephemeris.

## 6. The ten wewaran cycles (Ekawara..Dasawara)
With `p = pawukon_day` (1..210):

- **Ekawara (1)**: always **Luang**.
- **Dwiwara (2)**: `(urip_pancawara + urip_saptawara) mod 2` → 0=Menga, 1=Pepet.
- **Triwara (3)**: `(p-1) mod 3` → 0=Pasah, 1=Beteng, 2=Kajeng.
- **Caturwara (4) — semi-arithmetic**: names Sri, Laba, Jaya, Mandala/Menala. **Trijaya Dungulan**: Jaya runs 3 days in a row at p=71,72,73. Formula:
  ```
  if p ≤ 71: c = (p-1) mod 4
  elif p in {72, 73}: c = 2   // Jaya held
  else: c = (p-3) mod 4
  ```
- **Pancawara (5)**: see item 1.
- **Sadwara (6)**: `(p-1) mod 6` → 0=Tungleh, 1=Aryang, 2=Urukung, 3=Paniron, 4=Was, 5=Maulu.
- **Saptawara (7)**: same as `day_in_wuku = (p-1) mod 7`.
- **Astawara (8) — semi-arithmetic**: names Sri, Indra, Guru, Yama, Ludra, Brahma, Kala, Uma. **Kala held** for 3 days at p=71,72,73 (Babadbali: "Kala Tiga"). Formula (research output — note potential off-by-one to verify against kalenderbali.org spot checks):
  ```
  if p ≤ 70: a = (p-1) mod 8
  elif p in {71,72,73}: a = 6   // Kala held
  else: a = (p-3-1) mod 8       // research's formula
  ```
- **Sangawara (9) — semi-arithmetic**: Dangu repeats for days 1..4 then runs straight. Formula (research output):
  ```
  if p ≤ 4: s = 0   // Dangu
  else: s = (p-5) mod 9
  ```
- **Dasawara (10) — urip-derived**: `(urip_pancawara + urip_saptawara) mod 10` → 0=Pandita, 1=Pati, 2=Suka, 3=Duka, 4=Sri, 5=Manuh, 6=Manusa, 7=Raja, 8=Dewa, 9=Raksasa.

**Sanity check for 1985-05-05 (p=176, Redite Pahing)**:
| Cycle | Value |
|---|---|
| Ekawara | Luang |
| Dwiwara | (5+9)%2=0 → Menga |
| Triwara | 175%3=1 → Beteng |
| Caturwara | (176-3)%4=1 → Laba |
| Pancawara | Pahing |
| Sadwara | 175%6=1 → Aryang |
| Saptawara | Redite |
| Astawara | (176-3-1)%8=4 → Ludra |
| Sangawara | (176-5)%9=0 → Dangu |
| Dasawara | (9+5)%10=4 → Sri |

**Confidence**: High for pure-arithmetic cycles (Dwi/Tri/Sad/Dasawara). Medium-high for Catur/Asta/Sangawara — Babadbali documents the rules clearly, but spot-check 5-10 kalenderbali.org dates before shipping to catch off-by-one in the held-day formulas.

## Implementation roadmap

1. `src/jdn.ts` — Gregorian→JDN.
2. `src/wewaran.ts` — modular cycles + Dwi/Dasawara urip math.
3. `src/pawukon.ts` — wuku/pawukon-day + Catur/Asta/Sangawara semi-arithmetic.
4. `src/weton.ts` — pair pancawara+saptawara, compute neptu.
5. Vitest fixtures: the 4 anchor dates above plus spot-checks against kalenderbali.org.
6. v2: sasih, Saka, Nyepi, ngunalatri — needs ephemeris or precomputed lookup.

**Edge cases**: 1900-01-01 = JDN 2,415,021 (safely positive). Day boundary = civil midnight in user TZ; sunset/sunrise convention is a UI footnote.
