# Specs

Approved design specs, one file per feature/refactor:

```
YYYY-MM-DD-<topic>-design.md
```

A spec is written during brainstorming and **approved before any code** (see
[`WEB_BUILD_STRUCTURE.md`](../WEB_BUILD_STRUCTURE.md) §3). It captures: the problem,
2–3 approaches with the chosen one, the design in sections, and scope/non-goals.
Once approved, it drives an implementation plan in [`../plans/`](../plans).
